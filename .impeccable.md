## Design Context

### Users
Malaysian Chinese SME owners (bosses) attending Vibe Coding workshops. They are not developers — they are business operators learning to build software with AI tools (Cursor, v0, Claude). They arrive at the prompt library to find ready-to-use templates they can copy, customize with their business details, and paste into a Vibe Coding tool. Speed and clarity matter — they want to grab a prompt and go, not study documentation.

### Brand Personality
**Professional, Empowering, Approachable.** The trusted Vibe Coding authority in Southeast Asia. The tone is a knowledgeable mentor who respects the boss's time — confident but never condescending, practical over theoretical. The library should feel like a premium resource that makes students proud to be part of the community.

### Aesthetic Direction
**Clean & Structured.** Notion-like clarity with generous whitespace, strong typographic hierarchy, and well-defined cards. The interface should feel organized and scannable — a boss should find the right prompt in seconds.

**Key visual elements (from mockup):**
- White background with subtle card borders (no heavy shadows)
- Dark navy (#0F172A) code blocks for prompt content — the focal point of each page
- Yellow/gold (#FCD34D) as the primary accent — logo, CTAs, boss tip callouts
- Slate text for body copy, strong contrast for headings
- Pill badges for metadata (difficulty, time, version)
- Blue-tinted input fields for variable substitution
- Two-column layout on desktop, single column on mobile

**Typography:**
- Inter for Latin characters — clean, modern, professional
- Noto Sans SC for Chinese — excellent CJK readability
- Strong size hierarchy: large titles, medium subtitles, compact metadata

**References:** Clean marketplace/template gallery aesthetic (Notion templates, Shopify app store direction — merchant-friendly, not developer-intimidating)

**Anti-references — explicitly avoid:**
- Generic SaaS landing page templates (stock photos, cookie-cutter hero sections)
- Overly technical / developer-heavy aesthetics (no terminal UIs, no monospace-everything)
- Cheap or cluttered layouts (no ad-heavy feel, no information overload)
- Childish or gamified elements (no mascots, badges, excessive animations)

### Design Principles

1. **Clarity over cleverness.** Every element should be immediately understandable to a non-technical SME boss reading in Chinese. If it needs explanation, simplify it.

2. **Copy-first UX.** The entire interface exists to get a prompt into the user's clipboard. The copy button, variable inputs, and prompt preview are the hero elements — everything else supports them.

3. **Structured scanability.** Use cards, pills, and consistent metadata placement so users can scan a grid of 20+ prompts and find what they need without reading everything.

4. **Premium restraint.** Use whitespace generously, limit the color palette, and let typography do the heavy lifting. The yellow accent should pop precisely because it's used sparingly.

5. **Bilingual harmony.** Chinese is primary but English coexists in the schema. Design for CJK line lengths and character density — avoid layouts that only work with short English labels.
